EchoMem Agent Design

LoCoMo 评测台设计

在 echomem-agent 内增加 LoCoMo 数据集自动下载、样本详情查看、批量评测运行、实时对话与结果观察、历史评测回放能力。

设计文件：docs/design/agent/locomo-eval-design.html 数据源：snap-research/locomo data/locomo10.json 阶段：先设计，后实现
目标 数据集 效果页面 评测流程 接口 存储 实现计划

目标与边界

LoCoMo 评测台是现有 Agent Playground 的一个新页面，面向记忆能力调试。它不是替代聊天页，而是把固定基准数据转成可重复运行的任务：把长对话写入 EchoMemory，按 LoCoMo QA 标注提问，再把 Agent 回答与标准答案进行评分和归档。

必须支持

  • 首次进入页面或点击刷新时自动下载 LoCoMo 数据集到本地缓存。
  • 查看 sample、session、turn、QA、evidence、summary 等详情。
  • 选择一个或多个 sample / QA 发起评测。
  • 运行中实时显示回放对话、提问、Agent 回答、评分状态。
  • 按历史 run 查看总体分数、逐题结果、日志和复现实参。

建议支持

  • 支持 dry-run，只查看将写入 EchoMemory 的会话内容。
  • 支持继续失败 run，从未完成的 QA 开始恢复。
  • 支持导出 JSON / CSV，用于横向比较模型和 EchoMemory 配置。
  • 支持切换评分器：字符串包含、token F1、LLM judge。

暂不支持

  • 不下载 LoCoMo 图片，仅展示数据集中已有的图片 URL、caption 和查询词。
  • 不在第一版实现多模型并行竞赛。
  • 不修改 EchoMemory 内部存储结构，只通过 HTTP API 写入和检索。

LoCoMo 数据集接入

官方仓库说明 LoCoMo 是 ACL 2024 论文 "Evaluating Very Long-Term Conversational Memory of LLM Agents" 的数据和代码仓库。数据文件位于 data/locomo10.json，包含 10 个长对话样本，每个样本带有 conversation、observation、session_summary、event_summary 和 QA 标注。

评测流程

第一版采用可解释的串行评测流程，保证每一步都能在页面上观察和重放。运行粒度建议为 run -> sample -> session replay -> commit -> QA answer -> judge。

阶段动作实时展示失败处理
准备数据 确认本地 locomo10.json 可用；缺失则下载。 下载进度、文件大小、sha256、样本数量。 保留错误消息；允许用户改 URL 后重试。
创建 run 写入 run 配置：样本、QA 范围、模型、EchoMemory base_url、评分器。 run id、开始时间、配置摘要。 run 状态为 failed，不产生部分评分。
回放对话 按 LoCoMo session 顺序将 turn 写入 EchoMemory session。 当前 session、turn、speaker、dia_id。 记录最后成功 turn；支持 resume。
归档记忆 每个 sample 回放完成后触发 EchoMemory commit。 commit id、状态、抽取记忆摘要。 commit 失败时仍可选择直接 QA，但结果标记为 degraded。
执行 QA 逐题调用 Agent chat，问题作为当前请求发送。 模型上下文、检索项、Agent 答案。 单题失败可跳过并继续下一题。
评分归档 按选定评分器生成 exact、F1、judge verdict 和解释。 逐题分数、总体均值、类别分布。 评分失败不丢答案，允许稍后重新评分。

评分器

  • contains：标准答案文本被回答包含时记为 1，否则 0。成本低，但对同义表达不友好。
  • token_f1：参考常见 QA 评测，归一化后计算 token overlap。第一版默认启用。
  • llm_judge：把 question、gold answer、agent answer、evidence 摘要交给模型判定 correct / partial / wrong。需要额外模型调用。

Agent Server 接口设计

页面只调用 echomem-agent 自己的接口。agent server 内部继续通过 EchoMemory HTTP API 写入 session、commit 和检索，保持当前工程“不 import echomem 内部模块”的边界。

接口用途返回要点
GET /agent/locomo LoCoMo 评测页面。 HTML。
POST /agent/locomo/dataset/sync 下载或刷新数据集。 manifest、sample_count、downloaded。
GET /agent/locomo/dataset 读取样本索引和统计信息。 samples、categories、manifest。
GET /agent/locomo/dataset/{sample_id} 读取单个 sample 详情。 conversation、qa、summaries、events。
POST /agent/locomo/runs 创建并启动评测。 run_id、status、selected_items。
GET /agent/locomo/runs/{run_id} 轮询 run 快照。 status、progress、results、latest_event。
GET /agent/locomo/runs/{run_id}/events 实时事件流，优先 SSE。 dataset、replay、commit、qa、judge、done 事件。
GET /agent/locomo/runs 历史评测列表。 run id、时间、配置、总体分数、状态。

创建 run 请求示例

POST /agent/locomo/runs
{
  "sample_ids": ["sample_1", "sample_2"],
  "qa_ids": {
    "sample_1": ["qa_07", "qa_08"],
    "sample_2": "all"
  },
  "mode": "replay_then_qa",
  "scorers": ["token_f1", "llm_judge"],
  "commit_policy": "after_each_sample",
  "session_prefix": "locomo-eval",
  "dry_run": false
}

本地存储设计

LoCoMo 页面需要保存数据集缓存和历史结果。第一版可直接使用 JSON 文件，后续如果 run 数量变大再迁移到 SQLite。

文件内容用途
manifest.jsonsource_url、sha256、downloaded_at、sample_count。页面顶部数据源状态。
run.jsonrun 配置、状态、开始结束时间、模型与 EchoMemory 配置摘要。历史列表与复现。
events.jsonl实时事件追加日志。SSE 重连、失败定位。
results.json总体和逐题评分。结果页、导出。
contexts/*.json每题发送给模型的上下文和 trace。解释为什么答对或答错。

实现计划

1. 数据集服务

  • 新增 locomo_dataset.py：下载、校验、索引、读取 sample。
  • 新增配置：locomo.dataset_url、locomo.data_dir、locomo.auto_download。
  • 测试覆盖：首次下载、缓存命中、坏 JSON、结构缺失。

2. 评测服务

  • 新增 locomo_eval_service.py：run 生命周期、回放、commit、QA、评分。
  • 复用 AgentChatService 的上下文构造和 chat 调用。
  • 事件以 JSONL 落盘，同时推给 SSE。

3. 页面实现

  • 新增 /agent/locomo 页面，不影响现有聊天页。
  • 支持样本详情、QA 多选、运行视图、历史结果。
  • 先用轮询实现，SSE 作为增强路径。

验收标准

  • 本地无数据集时，打开页面可自动下载并显示 10 个 sample。
  • 选择一个 sample 的 1 个 QA 后可完成完整评测，并在页面看到回放、回答、分数和历史记录。
  • 失败 run 可在历史中打开，保留错误、已完成结果和最后事件。
  • 现有 /agent/chat、上下文检查器、EchoMemory 调试功能不回归。
