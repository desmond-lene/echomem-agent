3. 对 echomemory 的通用改造

3.1 Recall 编排增强（不改业务接口，增强内部计划）

  • 在 echomemory/src/echomem/recall/local_recall_orchestrator.py 增加 query-plan 后处理：按问题类型拆分子查询（time/entity/cause/task）。
  • 为 graph 引擎增加二次召回策略：第一次取证据，第二次按缺失槽位补召回（而非仅 broad retry）。
  • 新增去重与一致性约束：实体别名归一 + 关系方向约束，降低 M7 类矛盾。

3.2 领域模型补强（事件/因果/任务）

  • 新增 EventEdge（发生时间、参与者、结果）、CauseEdge（直接原因/根因）、TaskNode（目标/状态/依赖）。
  • commit 抽取阶段把“离职、融资、手术、团队拆分、邀请/拒绝”等模式结构化写入图，而非只落 episode 文本。

3.3 回答规划器（Answer Planner）

  • 在检索后新增规划阶段：先形成证据表（时间线/关系链/因果链），再生成回答。
  • 回答前做 contradiction check：同题内关键结论不允许互斥（如“同一公司/不同公司”冲突）。

3.4 可观测与回归

  • 保留每题 artifacts：query plan、召回项、证据链、最终回答、判分。
  • 把 Chenmo 场景纳入长期回归集，目标是阶段性提升而非一次性调参过拟合。

阶段目标主要改动验收
